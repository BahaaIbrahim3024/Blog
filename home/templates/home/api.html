Obtaining an Authentication Token

There're 2 ways to generate Authentication Token :

1- By Creating (Register) a new account

Request: POST

Url:

http://127.0.0.1:8000/api/account/register

Params:

email

username

first_name

last_name

age

gender

password (Your password)

password2 (retype your password to confirm it)


Success Response:

{
    "response": "Successfully registered anew user.",
    "email": "apiTest@yahoo.com",
    "username": "apiuser",
    "pk": 1,
    "token": "9720ee3080198d3c617bd61d5123bb3d6b3e332d"
}

2- By Login to your account

Request: POST

Url:

http://127.0.0.1:8000/api/account/login

Params:

username (Enter your login email)

password (Your password)

No that's not a typo above. The param is username but you use your email.


Success Response:

{
    "response": "Successfully authenticated.",
    "pk": 1,
    "email": "apiTest@yahoo.com",
    "token": "9720ee3080198d3c617bd61d5123bb3d6b3e332d"
}

Creating a Blog Post

Request: POST

Headers:

Authorization: Token <token>

Make sure that only user (Authorized user) can create posts so you must attached your token to header in the request


Url:

http://127.0.0.1:8000/api/blog/create

Params:

title (text)

body (text)

image (file)


Success Response:

{
    "response": "created",
    "title": "Test Title!",
    "body": "this is new body for test \ncool.\nstuff.",
    "image": "/media/blog/1/New_Title.png",
    "date_updated": "2021-02-27T15:19:33.095626Z",
    "username": "apiuser"
}

Viewing a Details for Blog Post

Request: GET

Headers:

Authorization: Token <token>

Url:

http://127.0.0.1:8000/api/blog/<slug>/

Success Response:

{
    "title": "Variance and Bias",
    "body": "this post about Variance and Bias and the relationship with Overfitting and Underfitting.",
    "image": "/media/blog/1/newPic.png",
    "date_updated": "2021-02-27T15:20:00.095626Z",
    "username": "apiuser"
}

Updating (edit) a Specific Blog Post

Request: PUT

Headers:

Authorization: Token <token>

Make sure that only the owner of this post who is editing the post so you must attached your token to header in the request


Url:

http://127.0.0.1:8000/api/blog/<slug>/update

Success Response:

{
    "response": "Update Successful"
}

Deleting (remove) a Specific Blog Post

Request: DELETE

Headers:

Authorization: Token <token>

Url:

http://127.0.0.1:8000/api/blog/<slug>/delete

Make sure that only the owner of this post who can delete the post so you must attached your token to header in the request


Success Response:

{
    "response": "Deleted Successful"
}

List Blog Posts (Pagination, Search and Ordering)

Request: GET

Headers:

Authorization: Token <token>

Url:

http://127.0.0.1:8000/api/blog/list/?search=<query>&ordering=-date_updated&page=1

Params:

query (this will search the title and body of blog posts for a match)

ordering (Can use: title, body, username or date_updated)

page (results will be shown per page.)


Success Response:

{
    "count": <num_results>,
    "next": null,
    "previous": null,
    "results": [
        {
            "title": "gfhhfghfgh",
            "body": "fghgfhfghf",
            "date_updated": "2021-02-27T21:43:56.452234Z",
            "username": "apiuser"
        },
        ...
}
